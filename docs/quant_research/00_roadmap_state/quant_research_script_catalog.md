This catalog is the first stop before running, moving, or adding quant-research scripts. It exists to keep roadmap execution from depending on filename memory or ad hoc search.

## How To Use

- Use `run priority` first: it answers whether a script is a default manual entrypoint, scheduler-only surface, quarantined falsification path, historical record, or supporting tool.
- Use `category` to pick the relevant lane before opening a script.
- Use `status` to distinguish active entrypoints from quarantined or historical research branches.
- Use `safe-to-move` before any path refactor. `no` means public entrypoint; `yes-with-wrapper` means preserve CLI compatibility if moved.
- This catalog is covered by `tests/test_static_contracts.py`; adding a `.py` or `.ps1` script without a row fails the static contract.

## Status Labels

- `active`: current or near-current data/research workflow.
- `scheduled_entrypoint`: public scheduled-task path; preserve exact path or wrapper.
- `supporting`: diagnostic, report, probe, exporter, helper workflow, or compatibility wrapper.
- `historical`: retained evidence for old roadmap branches or superseded variants.
- `quarantined`: preregistered, sidecar, or strict-falsification candidate not admitted to the canonical parent.
- `deprecated_candidate`: cleanup or legacy candidate path that should not be used as a new starting point.
