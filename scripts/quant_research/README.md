## Current Entry Contract

This README uses the same entry language as the roadmap state file:

| task type | entry document | script run priority | rule |
| --- | --- | --- | --- |
| current Binance PIT h10d hardening | [latest h10d validation](../../docs/quant_research/02_binance_pit_h10d/binance_canonical_h10d_validation_2026_05_12.md) | `default_entrypoint` | start here for active manual research work |

| quarantined falsification | preregistration or falsification report for that lane | `quarantined_falsification` | run only for that named quarantine test |
| historical audit or rediscovery | branch ledger row plus older report | `historical_do_not_start_here` | read for evidence; do not use as a new start point |
| one-off diagnostic/report writing | current roadmap/report that asks for it | `supporting_tool` | supporting tools require an explicit caller or doc reason |

Default manual research route:

```text
roadmap state
-> script catalog
-> default_entrypoint rows in canonical_h10d_and_binance_pit
-> latest Binance PIT h10d validation report
```

## Scheduled-only Entrypoints

`scheduled_only` scripts are registration and runner surfaces for Task Scheduler. They are listed row-by-row in the catalog and default to `safe-to-move = no`.

## Quarantined Falsification Scripts

`quarantined_falsification` scripts are preregistered sidecar or strict-falsification implementations. Compatibility wrappers for those scripts are cataloged as `supporting_tool`, not as separate falsification implementations.

## Historical / Do-not-start-here Scripts

`historical_do_not_start_here` scripts preserve old evidence, rejected branches, or superseded variants. Use them for audit and rediscovery, not for new default execution.

## Generated Catalog Contract

The catalog has one row per `.py` / `.ps1` script file under `scripts/quant_research` and records: script path, category, status, run priority, purpose, primary inputs, primary outputs, related roadmap/doc, and safe-to-move policy. Compatibility wrappers stay discoverable as `supporting_tool` rows while moved implementations keep their research lifecycle status. `tests/test_static_contracts.py` fails if a script is missing from the catalog or if enum values drift.
